## What it does

Emits a paid ad on an AI surface ‚Äî ChatGPT Ads today; Perplexity / Gemini / Grok / Copilot as adapters land. Engine-agnostic by design: the `engine` column on `paid_ad_objects` plus the per-engine adapter file is the whole abstraction.

Under the hood:

1. Persona-intent matrix validates the creative content (block-severity halts; warn-severity logs).
2. Calls the engine adapter to create the campaign / ad_group / ad on the external system.
3. Persists to `paid_ad_objects` and writes the Action‚ÜîOutcome Ledger row via Track A's bridge.

Intent is **always `'commercial'`** for `emit_paid` ‚Äî paid placements exist to drive conversions, never just engagement. The caller cannot override this.

## When to use

- The user (or upstream agent) has decided to launch a paid placement against a specific prompt cluster.
- `recommend_paid_ad_strategy` returned a proposal the user approved.

## When not to use

- The user wants to pause / activate / archive an existing object ‚Äî use `manage_paid_ad`.
- The user wants strategy proposals, not creation ‚Äî use `recommend_paid_ad_strategy`.
- The user wants reporting ‚Äî use `get_paid_ads_insights`.

## Inputs

- `object_type` (`'campaign'` | `'ad_group'` | `'ad'`, required).
- `engine` (`'chatgpt'` | `'perplexity'` | `'gemini'` | `'grok'` | `'copilot'`, default `'chatgpt'`) ‚Äî future engines drop in via adapter files.
- `payload` (object, required) ‚Äî engine-shaped payload for the create API call (e.g. ChatGPT Ads campaign body).
- `persona_id` (number, required) ‚Äî persona this placement targets; drives matrix validation.
- `content` (string, optional) ‚Äî creative body to validate. Strongly recommended for `'ad'` object_type.
- `parent_external_id` (string, optional) ‚Äî required for `'ad_group'` / `'ad'`; the parent campaign or ad_group external id.
- `citable_prompt_ids` (number[], optional) ‚Äî `s_prompts` ids this placement is meant to win.
- `citable_hub_id` (number, optional) ‚Äî `generated_content` id of the hub this ad amplifies (links spoke ‚Üí hub in the ledger).

## Response

```json
{
  "externalId": "campaign_abc123",
  "engine": "chatgpt",
  "objectType": "campaign",
  "payload": { "...": "engine-shaped echo" },
  "validation": {
    "status": "allowed",
    "violations": []
  }
}
```

On block: `isError: true` with `structuredContent: { status: 'blocked' | 'no_connector' | 'error', ... }`. For `'blocked'`, call `POST /api/persona-intent/:brandId/override` first and retry. For `'no_connector'`, connect the engine in `/connectors` first. The ledger row is written internally by Track A's bridge; inspect via `get_action_outcome_history` if needed.

## Persona-intent validation

**Required.** Track A's `createPaidObject` runs the matrix before the external adapter is called ‚Äî no external system gets a side effect for content the matrix blocks. Intent is hard-coded to `'commercial'`. Block-severity violations halt the emit unless an override is recorded out-of-band on the brand. See [Hard rules #2](/platform/hard-rules#2-every-emit-validates-persona-intent-before-emission).
