Marks a social reply as posted with the live URL. Sets status='published', stamps posted_url + posted_at. Brand resolved from MCP auth context.

Use this when:
- The user has just posted the reply on Reddit/Hacker News/etc. and wants it tracked.
- A user pastes a comment URL after replying manually.

Skip when:
- The reply hasn't been posted yet — use update_reply for status changes that don't carry a URL.
- The user wants to undo "posted" status — call update_reply with status='scheduled' or 'draft'.

The posted_url should be the canonical URL of the live comment, not the source thread URL — Citable uses it for citation attribution.

| Field | Type | Required | Default | Description |
|---|---|---|---|---|
| `reply_id` | `integer` | yes | — | ID of the reply row. |
| `posted_url` | `string` | yes | — | Canonical URL of the live comment (not the source thread URL). |
