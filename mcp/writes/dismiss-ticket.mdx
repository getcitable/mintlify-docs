Cancels a Citable GEO ticket with a required written reason. Use when the user has decided NOT to act on a recommendation and wants to clear it from their backlog — or when you are confident the recommendation is wrong for the brand and want to record why.

## Inputs

- `ticket_code` (required, e.g. `"CIT-052"`) — the Linear-style code from `get_geo_priorities` or the web UI.
- `reason` (required, 10–280 chars) — the "why." Examples: "Already handled manually last week", "Not aligned with Q2 focus on enterprise", "User decided to skip this channel entirely."

## Behavior

The reason is stored on the ticket row (visible on the web detail page) and in the MCP tool-call audit log. Cannot be changed via this tool afterwards — re-dismissing an already-canceled ticket returns the existing reason and does not overwrite.

Cannot dismiss a ticket that was already marked done — if the user wants to undo a completion, they do it in the web UI.

Returns the canonical deep link so you can cite the page.

## When NOT to use

Skip when: the user has actually completed the action (use `approve_ticket` instead).
