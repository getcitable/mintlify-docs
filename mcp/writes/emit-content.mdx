Emit a piece of content for the brand: validates it against the target persona's guardrails, saves a draft at status='queued', and records the action so its citation / share-of-voice impact can be measured over time.

This is the one-call way to record a complete content action. If you want staged control — draft now, schedule later, mark posted when it goes live — use create_content_draft / schedule_content / mark_content_posted instead.

Use this when:
- The user (or upstream agent) has decided WHAT to post and on WHICH channel.
- You want one atomic emit that records the action for later outcome measurement.
- You're building a hub-spoke fan-out: set fan_out_role='hub' on the long-form piece, then call emit_content again with fan_out_role='spoke' + parent_action_id=\<hub action id> for each channel adaptation.

Skip when:
- The user is brainstorming (no decision yet) — ingest_conversation instead.
- The content is a paid ad — use emit_paid.
- The action is an outreach DM/email — use emit_outreach.

Persona guardrails: persona_id + intent are REQUIRED — every content emit is validated against the target persona's guardrails. If a blocking violation fires, fix the body to clear it, or pass override_reason (≥ 8 chars) to proceed; overrides are recorded with your reason.

Inputs:
- persona_id (required): the persona this content targets.
- intent (required): 'engagement' (build relationships) or 'commercial' (drive a buy decision).
- platform (required): linkedin / twitter / reddit / blog / instagram / youtube.
- body (required, 1-8000 chars): the draft. Long-form for blogs; short-form for socials.
- title (optional): caps 200 chars. Defaults to body[:80].
- source_task_id OR from_conversation_id (one of, optional): anchor.
- action_subtype (optional): e.g. 'long_form_hub', 'reddit_spoke', 'linkedin_spoke'.
- fan_out_role (optional): 'hub' for the canonical piece; 'spoke' for adaptations.
- parent_action_id (optional): when fan_out_role='spoke', the hub action's id.
- target_prompt_ids (optional): integer[] of tracked-prompt ids this content targets (used to scope outcome measurement).
- override_reason (optional, ≥ 8 chars): bypass block-severity violations with audit.

Response: \{ actionId, contentId, deepLinkUrl, geoScore, artifactsFlagged, validation: \{ status, violations, overrideIds \} \}. (geoScore is the unified GEO/AEO citability score; call score_content for the explainable breakdown + per-dimension fixes.)
