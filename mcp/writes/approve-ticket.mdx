Marks a Citable recommendation ticket as done. Use after the user has completed the recommended action (published a piece of content, fixed a technical issue, and so on) or has explicitly asked you to approve the recommendation.

## Inputs

- `ticket_code` (required, e.g. `"ABC-051"`) — the code visible in the web app and returned by [`get_geo_priorities`](/mcp/reads/get-geo-priorities). The prefix matches your brand.
- `note` (optional, max 500 chars) — free-text reason for approval; stored in the audit log.

## Behavior

Idempotent: approving an already-done ticket returns the current state without re-writing. A dismissed ticket cannot be approved via this tool — it must be reopened in the web app first.

Returns the canonical deep link for the ticket so you can cite the page back to the user.

## When not to use

- The user wants to cancel or reject a ticket — use [`dismiss_ticket`](/mcp/writes/dismiss-ticket).
- The user wants details on a ticket they haven't identified yet — use [`get_geo_priorities`](/mcp/reads/get-geo-priorities) first.
