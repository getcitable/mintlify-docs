Marks a Citable GEO ticket as done. Use after the user has completed the recommended action (published a piece of content, fixed a technical issue, etc.) or has explicitly asked you to approve the recommendation.

Input:
- ticket_code (required, e.g. "GEO-124") — the ticket code visible in the web UI and returned by get_geo_priorities.
- note (optional, max 500 chars) — free-text reason for approval; stored in the tool-call audit log.

The tool is idempotent: approving an already-done ticket returns the current state without re-writing. A dismissed ticket cannot be approved via this tool — it must be reopened in the web UI first.

Returns the canonical deep link for the ticket so you can cite the page back to the user.

Skip when: the user wants to cancel/reject a ticket (use dismiss_ticket), or wants details on a ticket they haven't identified yet (use get_geo_priorities first).
