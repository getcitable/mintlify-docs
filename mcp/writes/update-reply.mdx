Updates a draft social-listening reply (Reddit, Hacker News, future Twitter/LinkedIn). Patches reply_text and/or status; the brand is resolved from the MCP auth context.

Use this when:
- The user has reviewed the auto-generated reply text and wants Claude to revise it.
- A draft needs to move to "queued" or "cancelled" without posting.

Skip when:
- The user wants to mark the reply as posted with a live URL — use mark_reply_posted instead.
- The user wants to schedule it for a specific date — use schedule_reply.
- The reply doesn't exist yet — replies are created via the Reddit/HN UI's Generate Reply button.

Allowed status values: draft, queued, scheduled, published, cancelled. Note that flipping to published without supplying posted_url leaves the URL blank — prefer mark_reply_posted for that transition.

Returns the updated reply on success, or an error message if the reply doesn't exist or doesn't belong to this brand.

## Inputs

| Field | Type | Required | Default | Description |
|---|---|---|---|---|
| `reply_id` | `integer` | yes | — | ID of the reply row to patch. |
| `reply_text` | `string` | no | — | New reply text. Plain text; the editor renders newlines as paragraph breaks. |
| `status` | `'draft' \| 'queued' \| 'scheduled' \| 'published' \| 'cancelled'` | no | — | New lifecycle status. To mark as posted with a URL, prefer mark_reply_posted. |
