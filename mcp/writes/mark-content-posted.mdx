Records that a content draft has been posted by stamping the live URL and marking status='published'. This is a RECORDING — Citable does not call platform APIs to post for the user. Workflow: Claude drafts → user reviews/edits → user posts manually on the platform → user (or Claude on user's instruction) calls this with the live URL.

Once stamped, the nightly attribution snapshot job picks up the post and starts tracking citation rate. Use get_attribution_report afterward to see lifetime / recent / trajectory metrics.

Use this when:
- The user has posted the content somewhere and is sharing the live URL.
- The user wants to retroactively log a post that's already live.

Skip when:
- The user has not yet posted the content (use schedule_content to keep it on the calendar).
- The URL is not http/https — every other scheme is rejected.

Idempotent: re-calling with the same content_id preserves the original posted_at via COALESCE; posted_url is overwritten so typos can be corrected.

## Inputs

| Field | Type | Required | Default | Description |
|---|---|---|---|---|
| `content_id` | `integer` | yes | — | ID of the content row. |
| `posted_url` | `string` | yes | — | Live URL of the posted content (must be http or https). |
| `posted_at` | `string` | no | — | Optional ISO timestamp of when the post went live. Defaults to now() on first call; preserved on idempotent re-calls. |
