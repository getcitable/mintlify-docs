## What it does

Emits an outreach action: validates persona-intent, creates or updates the social-reply row for the target thread, and records the action so its 30-day citation impact can be measured. Today this fully wires the social-listening channels (Reddit, Hacker News, X, LinkedIn listening). Cold email and LinkedIn DM channels return `status: 'pending_connector'` until those connectors are available â€” the action is still recorded so the intent appears in your history.

## When to use

- The user wants to reply to a Reddit / Hacker News / X / LinkedIn thread that's queued for response.
- An ingested conversation is ripe for a follow-up reply (pair with the `source_ref_id` of the thread).

## When not to use

- The outreach is broad content publishing â€” use [`emit_content`](/mcp/writes/emit-content).
- The outreach is a paid placement â€” use [`emit_paid`](/mcp/writes/emit-paid).

- `persona_id` (number, required) â€” persona the reply speaks as.
- `intent` (`'engagement'` | `'commercial'`, required) â€” `engagement` builds relationships; `commercial` pitches or converts.
- `channel` (`'reddit'` | `'hn'` | `'twitter'` | `'linkedin_listening'` | `'gmail'` | `'linkedin_dm'`, required) â€” first four are fully wired; last two return `pending_connector`.
- `reply_text` (string, 1â€“4000 chars, required) â€” body of the reply.
- `source_ref_id` (number, required for social channels) â€” the thread id this responds to.
- `source_url` (string URL, optional) â€” source thread URL for audit.
- `brand_account_id` (number, optional) â€” the account this reply posts as.
- `scheduled_for` (`YYYY-MM-DD`, optional) â€” lands on the calendar at a specific date instead of `'queued'`.
- `target_prompt_ids` (number[], optional) â€” tracked prompts the reply is meant to win citations on.

## Persona-intent validation

Outreach almost always touches a persona, so the validation matrix fires on every emit. Block-severity rules halt unless `override_reason â‰Ą 8 chars` is supplied. For `pending_connector` channels the matrix still runs, but a failure doesn't suppress the `pending_connector` response â€” you need to know which connector is missing more than you need an action record.
