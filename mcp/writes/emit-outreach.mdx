## What it does

Emits an outreach action: validates persona-intent, creates or updates the `social_replies` row for the target thread, and records to the Action‚ÜîOutcome Ledger for 30-day measurement. v1 fully wires the social-listening channels (Reddit / HN / Twitter / LinkedIn listening). Cold-email and LinkedIn DM channels return `status: 'pending_connector'` until Slice 3's Composio Gmail + LinkedIn connectors land ‚Äî the ledger row is still written so the intent shows in history.

## When to use

- The user wants to reply to a Reddit / HN / Twitter / LinkedIn thread that's been queued for response.
- An ingested conversation is ripe for a follow-up reply (pair with the `source_ref_id` of the thread).

## When not to use

- The outreach is broad content publishing ‚Äî use `emit_content`.
- The outreach is paid surfacing ‚Äî use `emit_paid`.

## Inputs

- `persona_id` (number, required) ‚Äî persona the reply speaks as.
- `intent` (`'engagement'` | `'commercial'`, required) ‚Äî `engagement` builds relationships; `commercial` pitches / converts.
- `channel` (`'reddit'` | `'hn'` | `'twitter'` | `'linkedin_listening'` | `'gmail'` | `'linkedin_dm'`, required) ‚Äî first four are fully wired; last two return `pending_connector`.
- `reply_text` (string, 1‚Äì4000 chars, required) ‚Äî body of the reply.
- `source_ref_id` (number, required for social channels) ‚Äî the Reddit/HN/Twitter/LinkedIn thread id this responds to.
- `source_url` (string URL, optional) ‚Äî source thread URL for audit.
- `brand_account_id` (number, optional) ‚Äî the `brand_account` this reply posts as.
- `scheduled_for` (`YYYY-MM-DD`, optional) ‚Äî lands on the calendar at a specific date instead of `'queued'`.
- `target_prompt_ids` (number[], optional) ‚Äî `s_prompts` ids the reply is meant to win citations on.
- `override_reason` (string, ‚â•8 chars, optional) ‚Äî bypass block-severity violations with audit.

## Persona-intent validation

**Required.** Outreach almost always touches a persona (your insider account, an audience persona, etc.), so the matrix fires on every emit. Block-severity rules halt unless `override_reason ‚â• 8 chars` is supplied. For `pending_connector` channels the matrix still runs, but a validation failure doesn't suppress the `pending_connector` response ‚Äî the caller needs to know which connector is missing more than they need a ledger row. See [Hard rules #2](/platform/hard-rules#2-every-emit-validates-persona-intent-before-emission).
