---

## What it does

Emits an operational recommendation — a config or setup change the brand should make to improve their Citable footprint: enable an additional AI model, add a journey stage, register a new `brand_account`, calibrate a persona, raise `scan_questions_per_stage`. v1 inserts an `action_engine_tasks` row and writes the Action↔Outcome Ledger row; the user reviews and accepts in the UI, which flips the underlying brand setting. Auto-apply is a follow-up once recommendation precision is proven.

Operational actions are persona-agnostic, so the persona-intent matrix is skipped — but the emit still routes through the same `validateAndRecordEmission` helper for ledger uniformity.

## When to use

- `get_brand_context` shows a coverage gap — e.g. only ChatGPT is enabled when competitors are also winning on Perplexity.
- The user asks "what should I set up next?"

## When not to use

- The fix is structural / on-page — use `emit_technical`.
- The fix is content creation — use `emit_content`.

## Inputs

- `subtype` (string, 1–40 chars, required) — free-form category. Common values:
  - `'enable_model'` — recommend enabling an additional AI model in `scan_models_enabled`
  - `'add_persona'` — recommend adding a new persona (audience subtype) to the matrix
  - `'add_competitor'` — recommend a competitor to track in `s_brand_competitors`
  - `'calibrate_persona'` — recommend a persona-calibration tune
  - `'expand_journey'` — recommend adding a stage to `scan_stages_enabled`
  - `'add_brand_account'` — recommend a `brand_account` / channel account to connect
- `title` (string, 1–200 chars, required) — one-line summary.
- `description` (string[], 1–20 items, each ≤500 chars, required) — step-by-step instructions.
- `effort_minutes` (number, 1–120, default 15) — estimated effort.
- `priority` (`'low'` | `'medium'` | `'high'` | `'critical'`, default `'medium'`).
- `config_target` (object, optional) — JSON describing the proposed setting change, recorded on the ledger row for later auto-apply. Example:
  ```json
  { "table": "brands", "column": "scan_models_enabled", "value_add": ["perplexity"] }
  ```

## Response

```json
{
  "ledgerId": 8720,
  "taskId": 4485,
  "deepLinkUrl": "https://getcitable.com/actions/4485",
  "validation": { "status": "allowed" }
}
```

## Persona-intent validation

**Persona-agnostic.** Operational edits don't speak as a persona, so `persona_id` and `intent` are not accepted and the matrix is skipped. The emit still flows through `validateAndRecordEmission` for ledger uniformity (one consistent contract per Hard rule #2) — the helper short-circuits when `personaId: null`. See [Hard rules #2](/platform/hard-rules#2-every-emit-validates-persona-intent-before-emission).

## Examples

Recommend enabling Perplexity because competitors are winning citations there:

```json
{
  "subtype": "enable_model",
  "title": "Enable Perplexity in scan_models_enabled",
  "description": [
    "Go to Settings → Scan Config.",
    "Check the Perplexity box in 'Models scanned'.",
    "Save. The next nightly scan will include Perplexity prompts.",
    "Review the new model's results in Brand Visibility after 24h."
  ],
  "effort_minutes": 5,
  "priority": "high",
  "config_target": {
    "table": "brands",
    "column": "scan_models_enabled",
    "value_add": ["perplexity"]
  }
}
```
