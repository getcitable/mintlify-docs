Returns the brand's identity snapshot — id, slug, plan tier, name, mission/description, industry, differentiators, personas (up to 10), distribution accounts per channel (up to 20) with voice/tone, recent brand-knowledge entries, and active competitors (up to 10). Use this to brief an AI on who the brand is before drafting content, reviewing positioning, or evaluating a competitor mention.

**Call this first.** Before any write (ingest_conversation, create_content_draft, mark_content_posted, etc.), call get_brand_context to verify the resolved brand matches what the user expects. Pay attention to `brand.planTier` and `brand.name` — when a user has multiple brands of the same name, planTier is the disambiguator. If the resolved brand looks wrong, call list_brands to see all brands the user has access to and surface the mismatch instead of writing.

Use this when the user asks:
- "Tell me about this brand."
- "What's my voice on LinkedIn?"
- "Who are my personas?"
- "What are my differentiators?"

Skip when the user wants metrics (use get_brand_visibility) or a to-do list (use get_geo_priorities).

Response shape: \{ brand: \{ id, name, slug, planTier, websiteUrl, description, industry, differentiators[] \}, personas[], accounts[], recentKnowledge[], activeCompetitors[] \}.

Takes no arguments — brand and user are resolved from the auth context.

## Inputs

_No inputs — call with an empty argument object._
