Returns a parent action ticket and all its child content tickets (drafts, scheduled, posted) in one call. Children are every piece of content created from this action. Each child includes status, scheduled_for, posted_url, assignee fields, and the canonical deepLinkUrl to its editor or detail page.

Use this when:
- The user asks about a specific action by ticket code ("Show me GEO-051 and its content").
- The user wants to know what content already exists against an action before drafting more.
- Claude is about to call create_content_draft and wants to confirm an action doesn't already have a queued draft.

Skip when:
- The user wants the brand-wide priority list (use get_geo_priorities).
- The user wants attribution for a specific posted URL (use get_attribution_report).

Input is a ticket code like "GEO-051". Brand is resolved from the MCP auth context — a ticket from another brand returns null even if the same sequence number exists in this brand.
