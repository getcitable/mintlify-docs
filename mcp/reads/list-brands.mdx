Returns every brand the authenticated user is a member of. Use this when:

- An agent has called get_brand_context and the resolved brand doesn't match what the user expected (different planTier, wrong workspace).
- The user references a brand by name and you need its id to retarget a write.
- The user has multiple brands with the same name (e.g. two "Acme Inc" entries — one trial, one paid) and disambiguation is required.

Read-only. Does not change the active brand — the MCP auth context is fixed for the session at connection time. To retarget writes to a different brand, create an API key under that brand in Citable (Settings → API keys), update the MCP config with it, and reconnect.

Response shape: \{ brands: [\{ id, name, slug, planTier, role, websiteUrl \}], activeBrandId \}. `activeBrandId` echoes the brand currently resolved by the auth context, so the caller can spot the active row.

Takes no arguments — user resolved from the auth context.

## Inputs

_No inputs — call with an empty argument object._
