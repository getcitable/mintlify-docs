Lists social-listening replies for the current brand. Filters by source platform (reddit, hn, twitter, linkedin_listening) and lifecycle status. Brand resolved from MCP auth context.

Use this when:
- The user asks "what Reddit replies are in draft?"
- Claude needs to find a reply to update before calling update_reply / mark_reply_posted.

Returns up to 50 most-recent rows (calendar surfaces the rest). Each row has id, source_type, source_ref_id, status, scheduled_for, posted_url, reply_text, brand_account_id.

## Inputs

| Field | Type | Required | Default | Description |
|---|---|---|---|---|
| `source_type` | `'reddit' \| 'hn' \| 'twitter' \| 'linkedin_listening'` | no | — | Filter by source platform. |
| `statuses` | `'draft' \| 'queued' \| 'scheduled' \| 'published' \| 'cancelled'[]` | no | — | Filter by status. Omit for all. |
