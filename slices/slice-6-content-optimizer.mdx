## What it does

Per page in `brand_website_content` (sitemap parse fallback when empty):

1. Run a Haiku-powered audit looking for 4 specific structural gaps.
2. For each gap found, emit an Action Engine recommendation via `validateAndRecordEmission`.
3. Write two linked rows: ledger emission + `action_engine_tasks` card.

V1 produces **recommendations only** — no CMS auto-apply. The user copies the suggested change to their own CMS. V2 will add Composio CMS-write connectors (WordPress, Webflow, Framer).

## v1 audit dimensions

Pure-content gaps detectable from text alone:

| Subtype | What it detects |
|---|---|
| `faq_schema_gap` | FAQ-shaped content (Q/A pairs, question headings) without FAQPage JSON-LD |
| `tldr_missing` | Long page (≥1500 chars) with no 2-3 sentence summary near the top |
| `eeat_signals_missing` | Missing ≥2 of: author byline, credentials, last-updated/published date |
| `comparison_block_missing` | "X vs Y" topic without a side-by-side comparison table |

Deferred to V2 (need richer signals than text alone): Information Gain vs competitors, canonical/duplicate fixes.

## Cost guards

- **Haiku 4.5** model — ~$0.001 per page audit
- `maxPages` default 20, **hard-capped at 100**
- Concurrency cap of 3 parallel LLM calls (max 5)
- **14-day dedup** on `(target_url × subtype)` — reruns are idempotent

## Sitemap fallback

If `brand_website_content` is empty for the brand:

1. Load `brand.website_url`, fetch `<url>/sitemap.xml`
2. Parse `<loc>` entries
3. **SSRF guard:** every URL must match the brand's hostname (off-domain links dropped)
4. Fetch each URL with 10s timeout, strip script/style/nav, cap at 16KB before LLM

If the sitemap parse fails, the audit completes with `pagesScanned: 0` — never throws.

## Route

```
POST /api/content-optimizer/audit/:brandId
Body: { maxPages?: number, skipDedup?: boolean }
Auth: brand_members active check (same as every internal route)
Returns: ContentAuditSummary
```

Synchronous run, 20-60s typical. For nightly cron usage this should later move behind a job queue.

## What's NOT shipped (production-risk callouts)

- ❌ Per-brand daily quota on auditor runs (~$0.02/run; quota belongs before customer-facing UX)
- ❌ Nightly cron registration (service is ready; not wired until manual runs validate quality)
- ❌ Async job queue (synchronous request blocks 20-60s; fine for admin, not customer UX)
